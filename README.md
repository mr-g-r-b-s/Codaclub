## вҡЎ What is CodaClub?

**CodaClub** is a real-time **1v1 multiplayer coding battle platform** where two developers race to fix intentionally broken code вҖ” under pressure, under attack, and against the clock.

Every round drops a new buggy function on both players. First one to identify and fix the bug scores points. Solve faster and you don't just pull ahead вҖ” you **steal time from your opponent**. Launch attacks to blur, flip, shake, or invert their editor. First to **500 points** wins.

> *Think LeetCode meets street fighter. Bug hunting as a contact sport.*

## рҹҢҹ Features

### рҹҸҶ Battle System
- **Real-time 1v1** вҖ” two players, one room code, instant sync via WebSockets
- **500-point win condition** вҖ” or most points when the 5-minute timer expires
- **Spectator Mode** вҖ” third party joins as live viewer, split-screen both players

### рҹ§  Challenge Pool
- **12 unique bugs** Г— **5 languages** = **60 total variants**
- Each round picks a **random language** вҖ” both players get the same one
- **JavaScript** uses real test execution in the browser
- **Python, Java, C++, SQL** use smart pattern matching to validate fixes

| Language | Validation |
|----------|-----------|
| рҹҹЁ JavaScript | Live test execution |
| рҹҗҚ Python | Pattern matching |
| вҳ• Java | Pattern matching |
| вҡҷпёҸ C++ | Pattern matching |
| рҹ—„пёҸ SQL | Pattern matching |

### вҡЎ Attack System
Earn points by solving вҖ” spend them to sabotage your opponent's editor:

| Attack | Effect | Cost |
|--------|--------|------|
| рҹҢ«пёҸ **Blur** | Blurs the entire editor | 50pts |
| рҹ“і **Shake** | Shakes the whole screen | 40pts |
| рҹҷғ **Flip** | Rotates everything 180В° | 80pts |
| рҹӘһ **Mirror** | Flips editor horizontally | 60pts |
| рҹҺЁ **Invert** | Inverts all colors | 30pts |
| рҹ”¬ **Tiny** | Shrinks the editor | 70pts |

### вҸұпёҸ Time Warfare
- 5-minute match timer displayed in the header
- Solve a round вҶ’ **+10 seconds** for you, **-10 seconds** for opponent
- Timer flashes рҹҹЎ yellow at 60s, рҹ”ҙ red at 30s
- Time runs out вҶ’ winner decided by most points

### рҹ”Ҡ Sound Effects
Fully synthesized via **Web Audio API** вҖ” zero audio files:
- рҹҺө Ascending ding on solve
- рҹ’Ҙ Glitch buzz when attacked
- рҹ”” Chime on bonus time
- рҹҘҒ Thud on time penalty
- рҹҺә Fanfare on victory
- рҹҳў Descending tones on defeat

## рҹҡҖ Getting Started

### Prerequisites
- Node.js 18+
- npm or yarn

### Installation

```bash
# Clone the repo
git clone https://github.com/yourusername/codaclub.git
cd codaclub

# Install server dependencies
npm install

# Install client dependencies
cd client
npm install
```

### Running locally

```bash
# Terminal 1 вҖ” start the battle server
node index.js
# рҹҡҖ CodaClub Battle Server on http://localhost:4000

# Terminal 2 вҖ” start the React client
cd client
npm start
# Opens http://localhost:3000
```

### Environment

Update the socket URL in `App.js` to point to your server:

```js
// App.js вҖ” socketSingleton
s = io('http://localhost:4000');
```

For production/ngrok, replace with your public URL and set the ngrok header if needed.

## рҹ”Ң Socket Events Reference

### Client вҶ’ Server
| Event | Payload | Description |
|-------|---------|-------------|
| `create-room` | `{ roomCode, playerName, challengeId }` | Host creates a room |
| `join-room` | `{ roomCode, playerName }` | Player 2 joins |
| `spectate-room` | `{ roomCode }` | Spectator joins |
| `send-attack` | `{ type, roomCode }` | Send attack to opponent |
| `round-won` | `{ roomCode, points, nextChallengeId, nextLanguage }` | Notify round solve |
| `my-points-update` | `{ roomCode, points, playerName, challenge, timeLeft }` | Sync live state |
| `match-won` | `{ roomCode, winnerPoints }` | Declare match winner |
| `time-up` | `{ roomCode }` | Timer expired |

### Server вҶ’ Client
| Event | Description |
|-------|-------------|
| `room-joined` | Both players connected вҖ” game starts |
| `receive-attack` | Attack payload from opponent |
| `opponent-round-won` | Opponent solved вҖ” move to next round |
| `time-penalty` | Deduct 10s from your timer |
| `match-over` | Opponent hit 500pts вҖ” you lose |
| `spectate-started` | Initial game state snapshot for spectator |
| `spectate-state-update` | Live state pushed to spectators |
| `opponent-disconnected` | Opponent left вҖ” you win |
